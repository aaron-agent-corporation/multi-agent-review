# Plan 05-07: Fix dist template copy destination

<objective>
Close the UAT Test-1 gap: the build copies `src/templates` to `dist/templates/`, but the compiled
resolver (`src/protocol/instructions.ts:22`, `new URL("../templates/...", import.meta.url)` from
`dist/src/protocol/instructions.js`) resolves `dist/src/templates/`. The compiled binary therefore
still ENOENTs at draft fan-out — the exact Phase-4 bug 05-01 was meant to fix. The 05-01 guard test
asserted existence at the wrong path (`dist/templates/`), so it passes while the binary is broken.
</objective>

<context>
Source: 05-UAT.md Gaps entry (test 1). Root cause fully diagnosed during UAT:
- `package.json` build: `tsc && node -e "...cpSync('src/templates','dist/templates',...)"` — wrong destination.
- tsc emits `dist/src/**` (rootDir includes src/ prefix), so the relative `../templates` from
  `dist/src/protocol/` is `dist/src/templates/`.
- Reproduction: `node dist/src/cli.js run input.md` against fake fixtures →
  `ENOENT ... dist/src/templates/agent-instructions.md.tmpl` (observed 2026-06-06, UAT run).
</context>
