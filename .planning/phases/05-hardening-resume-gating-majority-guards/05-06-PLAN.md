<objective>
The re-litigation guard vertical slice (RCRD-02 + the ledger/recording half of RSLV-03): the one
genuinely-new artifact — a rolling `runs/<id>/shared/resolved-decisions.md` ledger appended as forks
settle (response verdicts, convergence concessions, integrator calls, human rulings), readable by
agents as a peer artifact. The guard is inject + enforce (D-62): the seeded instruction file directs
agents to read the ledger before proposing changes (digest = one line per settled fork, D-65, keeping
prompts thin per D-37) AND the integrator/validation path drops a position reopening a settled
decision — drop + warn, no retry, logged `re-litigation` (D-64, generalizing 04-03's integrator drop).
The terminal decision record assembles FROM the ledger and sources the `resolver` field
(D-61/D-63/05-02 schema). The ledger WRITE reuses the injection-safe serializer; gray-matter stays
READ-only.

Purpose: RCRD-02 requires settled decisions to be fed forward as a guard so they are not re-litigated
within a run; RSLV-03 requires resolutions (including human rulings) recorded. This is the last slice
and depends on the majority resolver (05-03), the gated human ruling (05-05), and the ledger schema +
shared reader (05-02).
Output: the ledger writer + enforcement (resolved-decisions.ts), the seeded-file digest directive
(template), the append triggers + drop enforcement (engine.ts), the terminal record sourcing
(decision-record.ts), and inject/enforce/thin-prompt tests.
</objective>
