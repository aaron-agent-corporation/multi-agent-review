<objective>
The resume vertical slice (PROT-06): `mar resume <run-id>` / `mar resume --last` continues an
interrupted, failed, or paused run from the last completed phase by RE-DERIVING from the manifest
(D-14/D-54) ‚Äî read manifest ‚Üí find the first not-fully-satisfied phase ‚Üí rebuild the machine with
`initial` = that phase name + rehydrate context.roster ‚Üí run forward. No XState snapshot persistence
(Pitfall 2). Before continuing it re-validates (D-56): manifest integrity + every completed-phase
artifact exists and re-validates against its 04-01 schema (using the 05-02 shared tolerant reader) +
roster preflight, refusing with a specific error. The roster source differs by reason (D-57):
paused/interrupted ‚Üí survivors; failed/timeout ‚Üí full original roster. This slice also doubles as the
gated-approval continuation mechanism for 05-05's pause-and-exit (D-55).

Purpose: an interrupted run must resume without re-running prior phases or wasting completed work
(D-57 directly motivated by the Phase-4 live run that died at review below the vendor floor and wasted
three drafts).
Output: resumeProtocol + the manifest‚Üíresume-phase derivation + D-56 re-validation + the resume CLI
command + a fail-once fixture + resume e2e tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| on-disk completed-phase artifacts ‚Üí resumed run | the artifact trail is the resume input; tampered/corrupt artifacts are the tamper boundary re-validated before continuing |
| `<run-id>` CLI arg ‚Üí runs/ filesystem | a crafted id must not escape the runs/ tree |
| roster CLI auth ‚Üí resumed run | auth can decay between sessions (observed live with gemini) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering | a corrupted/altered completed-phase artifact silently carried into the resumed run | mitigate | D-56 re-validation re-parses every completed artifact against its 04-01 schema (tolerant reader) before continuing; specific refusal error (Task 1/3) |
| T-05-10 | Tampering | `<run-id>` path traversal escaping runs/ | mitigate | RUN_ID_RE validation reused before runDir resolution (Task 2) |
| T-05-11 | Denial of Service | resume re-derivation hangs on a restored mid-flight actor (XState snapshot footgun) | mitigate | re-derivation only ‚Äî no snapshot API in the resume path; resume entry is a phase name, interrupted phase re-runs from start (Pitfall 2) |
| T-05-12 | Spoofing | resume continues a run whose roster CLI auth has decayed, producing dropped-agent corruption | mitigate | runPreflight at resume time refuses with a per-agent failure (D-56) |
