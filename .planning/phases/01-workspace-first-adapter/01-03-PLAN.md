<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer CLI args → run id / prompt resolution | --run and --prompt values influence file paths and what gets spawned/logged |
| CLI → claude adapter | prompt content crosses into the spawned subprocess (mitigations owned by Plan 02 adapter) |
| turn outcome → manifest/log/console | what is persisted vs printed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Tampering | --run id used as a path segment | mitigate | When `--run` is supplied, resolve via `runDir(id)` and require the manifest to already exist (readManifest); reject ids that do not match the `newRunId` charset `[A-Za-z0-9_-]`/no path separators before touching the filesystem (no `..`/`/`). |
| T-01-11 | Info Disclosure | console + log content | mitigate | Console prints progress only (D-08); invocation log stores promptRef not full prompt (D-15). Inherits Plan 02 T-01-08. |
| T-01-12 | Availability | hung real claude call | mitigate | timeoutMs default 600000 passed to the adapter; adapter kills + reports timedOut (D-17, inherits T-01-07). |
| T-01-13 | Spoofing | success determination at CLI layer | mitigate | CLI branches on `turn.ok` from the adapter (which already enforces exitCode===0 AND is_error===false); CLI never re-derives success from exit code or subtype. |
</threat_model>
