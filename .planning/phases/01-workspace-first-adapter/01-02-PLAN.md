<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| orchestrator → claude subprocess | prompt content crosses into a spawned process; argv is attacker-influenceable if prompt is untrusted |
| claude stdout → normalizer | untrusted/variable JSON parsed and validated |
| invocation record → log file | what gets persisted to disk (audit trail) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | claude subprocess spawn (prompt as argv) | mitigate | execa passes args as an array — NO shell, never `shell:true`, never string concatenation. Prompt content cannot inject shell commands (RESEARCH Security Domain). |
| T-01-06 | Tampering / DoS | parsing claude stdout | mitigate | `ClaudeJson.safeParse` + `.passthrough()`; unparseable stdout becomes a graceful `ok:false` TurnResult, never a crash (Pitfall 2). |
| T-01-07 | Availability | hung claude invocation | mitigate | execa `timeout` + `forceKillAfterDelay` 5000ms → SIGTERM then SIGKILL; reported as `timedOut` (D-17, Pitfall 3). Verified by --hang fixture test. |
| T-01-08 | Info Disclosure | invocation log content | mitigate | Log prompt *reference* not full content (D-15); never log `ANTHROPIC_API_KEY` (Security Domain V2/V7). Phase-1 prompts are dev inputs; redaction of document content is Phase 5. |
| T-01-09 | Spoofing | trusting exit code or subtype as success | mitigate | Success requires exitCode===0 AND is_error===false; `subtype` is never read (it lies on auth failure — verified). |
</threat_model>
