<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process → filesystem (runs/) | Run id and artifact names become path segments; prompt/raw content written to disk |
| developer input → run id / seq | Values that compose deterministic file paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | layout.artifactName / run id path composition | mitigate | Run id from `newRunId()` is `[A-Za-z0-9_-]` (nanoid + numeric date) only — no `/` or `..`; seq is a number zero-padded; agent constrained to known literal "claude" this phase. No user-supplied path segment reaches the filesystem unsanitized. |
| T-01-02 | Availability | manifest.json write | mitigate | Atomic temp+rename (D-16) — a crash mid-write leaves the prior complete manifest, never a corrupt one. |
| T-01-03 | Availability | artifact write | mitigate | Same temp+rename for artifact + raw sibling; "done" = exists AND non-empty so a half-written file is never counted complete. |
| T-01-04 | Info Disclosure | runs/ in git | mitigate | `.gitignore` includes `runs/` from scaffold (Pitfall 6) — prompt/output content never committed. |
| T-01-SC | Tampering | npm installs (execa/zod/commander/pino/fs-extra/nanoid + dev) | accept | All packages are named in the project's authoritative STACK.md/CLAUDE.md and version-verified against npm registry in RESEARCH Package Legitimacy Audit; no `[ASSUMED]`/`[SUS]`/`[SLOP]` packages → no install checkpoint required. |
</threat_model>
