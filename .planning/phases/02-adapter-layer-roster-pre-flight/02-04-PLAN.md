<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| probe invocation -> live vendor CLI | spends ~1 tiny invocation per agent (cost surface) |
| preflight cache (.mar/preflight.json) -> run-start gating (Phase 3) | a poisoned/stale cache could let an unhealthy roster start |
| CLI --version / stderr -> hint text | untrusted CLI output rendered into hints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Denial of Service / $$ | probe prompt cost abuse | mitigate | probe is a SINGLE tiny "pong" invocation with retries:0 + ~30s timeout; never loops; cached ~10min so back-to-back runs don't re-probe (D-27/D-33) |
| T-02-14 | Tampering | preflight cache poisoning | mitigate | cache validated by PreflightCache.parse on read; TTL (~10min) bounds trust window; atomic temp+rename write prevents partial/corrupt cache; lives outside runs/ (not run lineage) |
| T-02-15 | Info Disclosure | hints rendering CLI stderr | mitigate | hints are FIXED actionable strings keyed off failure class; env vars NAMED never their values; raw stderr not echoed verbatim into the cache hint |
| T-02-16 | Elevation of Privilege | probe sandbox | mitigate | probe uses the same read-only adapter argv (codex `-s read-only`, gemini no `-y`) — no write/edit perms during a probe |
| T-02-SC | Tampering | npm installs | mitigate | none added this phase (D-35) |
</threat_model>
