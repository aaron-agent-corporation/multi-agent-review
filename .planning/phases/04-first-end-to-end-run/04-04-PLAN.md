<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| evaluation-round artifacts → convergence guard | agreement signal is agent-authored; read+validated from disk, not trusted as prose |
| non-base drafts → integrator | proposed additions are peer-authored; integrator reviews before patching |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | Spoofing | redundant merging by multiple agents | mitigate | exactly one integrator (base author, D-44); integration gate expects 1 writer (Pitfall 4); other agents never merge |
| T-04-11 | Tampering | auto-merge of an unreviewed/conflicting addition | mitigate | integrator emits a per-addition verdict and may reject-conflicts-with-resolved BEFORE patching (REVW-05); never auto-merge (case study #1 anti-pattern) |
| T-04-12 | Denial of Service | unbounded convergence loop | mitigate | convergenceCap (default 10, D-41) hard backstop; cap-reached → escalate, never spin |
| T-04-13 | Repudiation | unlogged resolution decisions | mitigate | every integrator verdict + convergence concession carries a rationale threaded to the decision record (RSLV-01, written in 04-05) |
</threat_model>
